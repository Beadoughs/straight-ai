# Discovery Baseline

## Current Stack

- Framework: Next.js App Router (`next@16`).
- Language: TypeScript + React 19.
- Styling: Tailwind CSS 4 with shared UI primitives.
- Deployment target: likely Vercel (includes `@vercel/analytics`).

## Existing User Flow

1. User lands on homepage (`/`).
2. User reads value proposition and sections (benefits, comparison, testimonials).
3. User is directed to booking page (`/booking`).
4. Booking page renders Calendly inline widget for consultation.

## Gaps Identified

- Calendly URL is still a placeholder.
- Chatbot script integration is commented placeholder.
- No explicit public launch checklist or operating runbook.
- No documented MVP scope and acceptance criteria.

## Immediate Priorities

- Lock MVP goals and must-have experience.
- Replace placeholders with environment-driven config.
- Add operational docs and launch readiness checklist.
